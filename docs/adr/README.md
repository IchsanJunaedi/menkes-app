# Architecture Decision Records

This directory contains Architecture Decision Records (ADRs) for the SehatKu EHR system.

ADRs will be added during Phase 5 documentation, covering:

- Why NestJS over Express
- Why Keycloak over custom auth
- Why Turborepo for monorepo management
- Why FHIR R4 schema from day one
- Why WatermelonDB for mobile offline

## Format

Each ADR follows the template: `adr-NNN-title.md`
